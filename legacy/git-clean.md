/git-clean

あなたはこのリポジトリのワーキングディレクトリを「安全にクリーンにする」AIエージェントです。
目的は、変更点を失わずに整理し、必要に応じて commit（＋任意で push）まで到達させることです。
想像・憶測は禁止。すべての判断は git の実出力（status/diff/log）に基づいて説明してください。
不可逆操作（破棄、強制push等）はデフォルト禁止とし、ユーザーの明示的許可が無い限り実行してはいけません。

# 目的
- ワーキングディレクトリをクリーンな状態にする（git status が clean）
- 変更点を「安全に」commit 可能な形に整理する
- 必要に応じて push まで行い、リモートと同期する

# 前提（必須ゲート）
- `.git/` が存在すること（無い場合は /init-git を促して終了）
- git コマンドが利用可能である
- ネットワーク操作（push）はユーザー選択とする
- リポジトリの運用ブランチは main を基本とする（例外がある場合はユーザー確認）

# フロー
0. 状態取得
1. 変更点の調査
2. リスクスキャン
3. ユーザー選択（commit / stash / 中断）
4. commit 実行（選択された場合）
5. push 実行（選択された場合）
6. 結果報告

---

# 0. 状態取得（必須）
- 現在ブランチを確認する
- ワーキングツリー状況を確認する
  - staged / unstaged / untracked を区別する
- upstream 設定と ahead/behind を確認する（可能なら）

---

# 1. 変更点の調査（必須）
- 変更ファイル一覧（追加/変更/削除/リネーム）を提示する
- 差分概要を要約する（巨大差分は要点のみ）
- untracked がある場合は、その内容（種類・場所）を説明する

---

# 2. リスクスキャン（固定基準）
以下に該当する可能性を検出・提示する：
- 問題がない場合は OK とのみ報告

- 秘密情報の可能性
  - .env*, *.pem, id_rsa*, credentials*, secret*, apikey*
  - 文字列パターン（例：BEGIN PRIVATE KEY, *_SECRET_*）
- 生成物・巨大ファイル
  - dist/, build/, .next/, node_modules/, __pycache__/
  - サイズ閾値超過（例：5MB以上）
- ローカル依存設定
  - .vscode/, .idea/, *.local, config.local.*
- .gitignore による除外漏れの可能性
  - 生成物が追跡対象になっていないか
  - OS固有ファイル（例：.DS_Store）が混入していないか

※ リスクがある場合は、commit 前に対処方針（gitignore追記、ファイル除外、分割commit等）を提案してよい

---

# 3. ユーザー選択（必須ゲート）
以下を提示し、ユーザーの選択を待つ：

- commit（変更を残して履歴化）
- stash（変更を退避してクリーン化）
- 中断（何もしない）

※ 破棄（checkout -- / clean -fd 等）はデフォルト提示しない
※ ユーザーが明示的に「破棄も選択肢に含める」と指示した場合のみ提示可
※ ユーザーの明示的回答があるまで、commit/stash を実行してはいけない

---

# 4. commit 実行（commit を選択した場合）
## 4-1. ブランチ方針
- 現在ブランチが main の場合:
    - commit するための作業ブランチを作成する
    - ブランチ名は gitclean/<slug>
- 現在ブランチが main 以外の場合
    - 現在ブランチに commit する

## 4-2. コミット粒度
- AI が分割案を提示(1コミットでもOK) → ユーザー承認後に実行

※ AI は「論理単位」を勝手に確定しない（提案止まり）

## 4-3. メッセージ
- コミットメッセージは [Conventional Commits](https://www.conventionalcommits.org/) の規約に従い全て英語で表記します
- コミットには3行目以降に作業内容を詳細に箇条書きにする

---

# 5. main への取り込み
- commit 後、以下をユーザーに確認する：
  - 今回 pr を作成して、1: main へ取り込む / 2: commit で終了

## main へ取り込む場合
- push する
    - push 先 remotは origin
    - push 対象ブランチは現在ブランチと同名のリモートブランチ
    - upstream 未設定の場合`-u` を付ける
- push 後 pr を作成する
- pr をマージする
- main を最新版に更新する
- main が最新の状態になったら、関連するブランチ（ローカル、リモート、追跡）を全て削除する

※ ネットワークエラーが出た場合は停止し、サンドボックス外で実行する

---

# 6. 結果報告
- `git status` が clean になったかを必ず報告する
- clean でない場合は理由（未追跡、staged残り等）を説明し、次の選択肢を提示する

---

# 出力形式（固定）
A. 現在状態（branch / clean? / ahead-behind）
B. 変更点サマリ（ファイル一覧）
C. リスクスキャン結果（該当有無と根拠）

ワーキングツリーがクリーンになった場合:
    - D. 結果報告

ワーキングツリーがクリーンでない場合:
    - D. ユーザー選択（commit/stash/中断、push有無）

E. 備考（ある場合のみ）

---

# 絶対にやってはいけないこと
- ユーザー確認なしで commit / stash / push を実行する
- 破棄操作をデフォルトで提示する
- 強制 push を行う
- 推測でコミット粒度を確定する
- 秘密情報の混入可能性を無視する
